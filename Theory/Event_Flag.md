## 1. 개요 

- **정의**: 여러 개의 비트(Flags)를 하나의 그룹으로 묶어 태스크 간 동기화에 사용하는 객체이다.
    
- **구조**: `configUSE_16_BIT_TICKS` 설정에 따라 하나의 그룹당 **8비트** 또는 **24비트**의 이벤트 플래그를 가질 수 있다.
    
- **핵심 용도**: 여러 사건(Event)이 동시에 만족되어야 하거나(AND), 여러 사건 중 하나라도 발생했을 때(OR) 태스크를 실행시켜야 하는 복합 동기화 상황에 사용한다.
    

## 2. 주요 특징 및 메커니즘

- **다중 조건 대기 (AND/OR Logic)**:
    
    - **AND**: 설정한 모든 비트가 1이 되어야 대기 상태 해제 (예: 모든 드라이버 로드 완료 후 실행).
        
    - **OR**: 설정한 비트 중 하나만 1이 되어도 대기 상태 해제 (예: 여러 버튼 중 하나만 눌려도 실행).
        
- **방송(Broadcasting)**: 세마포어와 달리, 조건이 만족되면 해당 이벤트를 기다리던 **모든 태스크**가 동시에 깨어날 수 있다.
    
- **자동 클리어 (Auto-Clear)**: 태스크가 대기 상태에서 깨어날 때, 확인한 비트들을 자동으로 0으로 되돌리는 설정을 할 수 있다 (`xClearOnExit`).
    

## 3. 주요 API 요약

| **함수명**                     | **역할**       | **주요 특징**                          |
| --------------------------- | ------------ | ---------------------------------- |
| [xEventGroupCreate()](../FreeRTOS_reference_Manual/xEventGroupCreate().md)   | 이벤트 그룹 생성    | 힙 메모리 할당, 핸들 반환. ISR 호출 불가.        |
| [xEventGroupSetBits()](../FreeRTOS_reference_Manual/xEventGroupSetBits().md)  | 특정 비트를 1로 설정 | 비트 설정 즉시 대기 중인 태스크 확인. ISR용 별도 존재. |
| [xEventGroupWaitBits()](../FreeRTOS_reference_Manual/xEventGroupWaitBits().md) | 특정 비트 대기     | AND/OR 조건 설정, 타임아웃 지정 가능.          |
| [vEventGroupDelete()](../FreeRTOS_reference_Manual/vEventGroupDelete().md)   | 이벤트 그룹 삭제    | 메모리 해제, 대기 중인 태스크는 0을 반환하며 깨어남.    |

## 4. 세마포어 vs 이벤트 플래그 비교

|**구분**|**세마포어 (Semaphore)**|**이벤트 플래그 (Event Group)**|
|---|---|---|
|**신호 개수**|단일 신호|**다중 신호 (비트 단위)**|
|**대기 조건**|단순 발생 여부|**조합 조건 (AND/OR) 가능**|
|**수신 대상**|한 번에 하나의 태스크만|**여러 태스크 동시 수신(Broadcasting)**|
|**자원 소모**|신호마다 객체 필요|**하나의 그룹으로 여러 신호 관리**|
